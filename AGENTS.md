# AGENTS.md: Urekha AI Operating Guidelines

## 1. System Overview
Urekha is a dual-model Jyotish platform generating personalized Kundlis. AI agents operating on this codebase must adhere strictly to the separation of concerns across three primary domains: Online Users, Offline Agency, and Administration.

## 2. Agent Personas

### The Logic Architect (Gemini)
* **Role**: Data Synthesis & Prediction Engine.
* **Responsibilities**: 
    * Process raw `jyotishamastro` API payloads (Lagna, Charts, Dosha, Dasha).
    * Generate personalized, empathetic astrological predictions based on API data.
    * Ensure logic isolation between tenant types (User vs. Agent vs. Admin).

### The UI Craftsman (Claude)
* **Role**: Frontend Implementation & Styling.
* **Responsibilities**: 
    * Build the multi-dashboard architecture (User Dashboard, Agent Portal, Admin Console).
    * Implement print-perfect CSS (`@media print`) for offline Kundli generation.
    * Build strict, separate authentication guardrails for admins.

## 3. Domain Contexts & Rules
When generating code, agents must align with the target domain's specific requirements:

| Domain | Authentication | Core UI Focus | Priority |
| :--- | :--- | :--- | :--- |
| **Online User** | Standard Auth (Subscription gated) | Interactive Kundli, Calming Aesthetics | Responsiveness, Subscription Validation |
| **Offline (Agents)**| Internal Auth (Admin managed) | High-speed data entry, Print-ready PDF formats | Utility, Layout Precision (A4 formats) |
| **Administration**| Strict RBAC (Separate Auth) | Agent auditing, User management, System health | Security, Data Density, Access Control |

## 4. Core Integration Pipeline
3. Render: Output the combined data via React components (for web viewing) or specialized PDF templates (for agent printing).

## 5. Project Mapping & Synchronization (Codemap)
The `codemap/` directory contains JSON files that provide an exhaustive, search-optimized map of the project's architecture.

### Guidelines for Usage
* **Research Phase**: Before starting any task, agents MUST consult `codemap/root.json` to identify relevant folders and then drill down into specific sub-maps (e.g., `app.json`, `lib.json`).
* **Token Efficiency**: Use the codemap to understand folder structures and intents without repeatedly running expensive `ls` or `dir` commands.
* **Mandatory Synchronization**: If an agent creates, renames, or deletes any file or folder, they MUST immediately update the corresponding `.json` file in `codemap/` to reflect the change. This ensures the map remains a reliable source of truth.

## Golden Rule

**Always prefix commands with `rtk`**. If RTK has a dedicated filter, it uses it. If not, it passes through unchanged. This means RTK is always safe to use.

**Important**: Even in command chains with `&&`, use `rtk`:
```bash

# ✅ Correct
```
